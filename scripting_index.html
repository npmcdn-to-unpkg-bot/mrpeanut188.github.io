Creating the script file

Scripts are contained inside of .GSC files. To create a script, make a new .GSC file in your mod's maps folder. To edit an existing script, copy it from raw/maps to your mod's maps folder. Don't ever edit any files inside the raw directory unless you know what you are doing. Tick the checkmark next to the file in the mod tab of Launcher, which will include them in your IWD file when you build it.


Script Contents

Scripting is a lot like math. Math in real life is the reverse engineering of events and logic.
If you have 5 bottles and 2 are taken away, you have 3. Math is, in a sense, the reverse engineering of reality. We can apply it's concepts to simulate events that can happen.

As your learning of math has been built upon many different concepts, so must your coding knowledge. To actually understand the best ways of applying scripting to real-life concepts, you ARE going to have to read a lot.

For this first demonstration, we're going to do an idea generously donated by my guinea pig RadimaX: 

An in-game trigger that gives you 250 points and has a cooldown of 5 rounds.

To break this down, you have to think like a computer. You probably have heard computers work in 1's and 0's. So does logic: We can interpret this as true (1) and false (0), turning it into an if-then statement.

For example: 

IF player knifes a trigger AND it's been more than 5 rounds THEN give them 250 points.


This example will require some advanced techniques that are talked about more in-depth in their own dedicated sections.

To start please create a new script file following the directions at the top of the page. I am going to call mine knife_reward.gsc. Like said, scripting is like math. However, the scripting we are doing is going to be function based.

Let's look at a math function: 
f(x) = 2x + 5

Now let's turn that into a scripting function: 
f(x)
{
	return 2x+5;
}

In mathematical terms, these are the same, only formatted differently.
